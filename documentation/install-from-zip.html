Install from ZIP

Are you ready to get started with Respond?  Learn how to install it on a traditional LAMP deployment using a ZIP file.  Read more about the server requirements.

1. Create the directory structure for Respond

You create your directory structure by navigating to your web root and creating folders for Respond.

// go to your HTML root
cd var/www/html

// create the respond directory
mkdir respond

// switch directories
cd respond

// create a directory to hold the app and sites
mkdir app

// change to the app directory
cd app

// make sites directory in app (app/sites)
mkdir sites

2.  Make the sites directory writeable

This is typically done by making Apache the owner of the directory.  You can also set permissions on the folder.  We recommend setting a permission of 755 if you cannot set an owner.  Contact your hosting provider about the correct permissions for secure, writeable directories.

chown -R www-data sites

3.  Map domains

Respond cannot be installed using sub-directories.  We recommend you setup a domain for your application and a sub-domain for each of your sites.  This allows you to secure your application using https and provide nice URLs for sites created using the CMS.  Domains are typically setup using DNS and your htdocs file. Refer to your hosting provider's documentation on setting up domains.  For installs that will host multiple sites, we have created a guide for installing Wildcard DNS based on our Digital Ocean setup. 

.../respond
          .../respond/app		                -> http://app.myrespond.com
          .../respond/app/sites/mysite           -> http://mysite.myrespond.com

4.  Download ZIP file

Download the ZIP file from Github.  Just go to the project home and select Download ZIP.  It is in the bottom of the right column.  Here is the link for easy access.  

Unpack the file in your app directory.  Your structure should look like the structure pictured on the right with the contents of the ZIP file in the app directory along with the sites folder.

Click to Enlarge

5.  Create your Database

Respond using MySQL to store data for the CMS.  You need to install the schema /schema.sql to get it up and running.  Refer to your host documentation on installing and configuring MySQL.  

6.  Update Settings

Update the DB_NAME, DB_USER, DB_PASSWORD, and APP_URL in api/setup.php.  Your CORS constant should contain the URL to your sites.

/*********************************/
/*  BASIC SETUP                  */
/*********************************/

// DB connection parameters
define('DB_HOST', 'localhost');
define('DB_NAME', 'respond');
define('DB_USER', 'dbuser');
define('DB_PASSWORD', 'dbpass');

// APP URL
define('APP_URL', 'http://app.myrespond.com');

8. Create your first site

Simple go to your application URL (e.g. http://app.myrespond.com) and follow the instructions.  Your default passcode is iloverespond.
