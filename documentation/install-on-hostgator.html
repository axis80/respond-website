Install on Hostgator

Are you ready to get started with Respond?  Learn how to install it on a traditional LAMP deployment using a ZIP file.  Read more about the server requirements.

1. Create the directory structure for Respond

Use file manager provided to you in cPanel.

Create a folder called Respond in public_html.

2.  Upload and Extract ZIP

Download the ZIP file from Github.  Just go to the project home and select Download ZIP.  It is in the bottom of the right column.  Here is the link for easy access.  

Use File manager to Extract the Zip into the Respond directory.

After extracting the zip, rename the new folder to app.

3. Create the Sites Folder

Create the sites folder in the respond/app directory.  The result should be respond/app/sites.

Make sure the permissions are 755.  You can do this by right clicking on the folder.

4.  Create the Database

Use the MySQL Databases function of cPanel to create a new database.

Then add the schema using PHPMyAdmin.  You can find the schema at the root of the project on Github under schema.sql.  Here is a link for quick access.

3. Map a sub domain for the application

Map a subdomain (e.g. app.gator.respondcms.com) to the app folder you created earlier.

6.  Update Settings

Using the code editor in File Manager update the /respond/app/setup.php file.  Update the DB_NAME, DB_USER, DB_PASSWORD, and APP_URL constants in setup.php.

8. Create your first site

Simple go to your application URL (e.g. http://app.myrespond.com) and follow the instructions.  Your default passcode is iloverespond.
