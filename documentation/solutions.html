Solutions to Common Problems

This page contains a list of solutions to problems that come up when installing Respond.

I only see errors (red x's) when I try to create my site and the themes are not displayed.

This occurs because your API is not working. This is most likely due to mod_rewrite not working on your server. But, it also can be caused by other errors, such as your schema being out-of-date from your API, a missing dependency, or an error on the server. To test what is happening, navigate to http://path-to-my-api/site/test. If the API is working, you will see "API works!" in your browser. See the demo.respondcms.com test at https://demo.respondcms.com/api/site/test. A PHP error will indicate that you might have a problem with your schema or other error in the API. A 404 (not found) or a redirect to another page will indicate that you probably don't have mod_rewrite configured properly.

A site is created, but I cannot login to the application.

First, try to create another site and make sure you have the correct password. If it still does not work, chances are that Apache is not passing the Authorization header to the PHP handler. Stackoverflow has a solid solution to this problem:

http://stackoverflow.com/questions/18424055/authorization-header-and-apache-request-headers-function

You can also view the discussion on the topic:

https://groups.google.com/forum/#!topic/respond-cms/WdA7Y7VOc-A

I cannot get Respond running in a subdirectory

By default, the Respond application and sites created using Respond run in HTML5 mode. For Apache, this means that both cannot run in a subdirectory due to how the .htaccess files are configured. So, you need to install the app and the sites in subdirectories. For local installations, this means creating Virtual Hosts for your local installation. For server installations, you must create a subdomain for your app and a subdomain for each site you create in Respond. To make this easier, you can configure wildcard subdomains to automatically create subdomains for new sites.
