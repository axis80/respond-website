Creating Themes

This tutorial will provide you with everything you need to know about themes in Respond. This tutorial includes the location of themes, creating a new theme, and a description of all the files that make up a theme.

Sites as Themes

To build a theme for Respond 6, simply build a site and then add a theme.json configuration file to tell the CMS what containers can be edited and what elements can be dragged.

Theme Settings

The settings.json file should be placed in the .private/settings.json folder for your theme.

[
    {
        "id": "packages",
        "value": "core, bootstrap"
    },
    {
        "id": "editable",
        "value": "[role=\"main\"]"
    },
    {
        "id": "sortable",
        "value": ".col, .column"
    }
]
Setting Description
packages A comma separated list of built-in plugin packages (e.g. packages)
editable A comma separated list of selectors to elements that are editable
sortable A comma separated list of selectors to elements that are sortable. These should be a container of relative placed elements.

Default Content

Default content is the content added to a new page created within Respond. Default content should be placed in the .default.html file in the root of the theme.

Theme Best Practices

  1. Elements in the draggable containers should be positioned relative.
  2. Elements in the draggable containers should not be floated.
  3. An element should have a role="main" attribute. This is used as the main content for the page by the respond-content and respond-list plugins.

Installing a theme

Simply place the theme folder in resources/themes
