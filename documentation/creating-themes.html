Creating Themes

This tutorial will provide you with everything you need to know about themes in Respond. This tutorial includes the location of themes, creating a new theme, and a description of all the files that make up a theme.

Location and Installation

Themes for Respond are located in the /themes folder off your installation directory. By default, Respond ships with 1 theme: simple located at /themes/simple.

Creating a Theme

The easiest way to create a theme for Respond is to copy the simple theme to a new folder within the themes folder, and then use that as a starting point for your own theme. This tutorial will get into the details of what each theme file does.

Theme Folders

The following folders are used to create a theme for Respond.

Folder Description
/themes/theme-name/files Holds any files you want added to the site along with your theme. You can use this folder to upload JS and images that support your theme. These will be transferred to the sites files/ directory after installation. All files uploaded here will show up in the files section of the app and can be accessed by the user. For files that you don't want the user interacting with, consider using the resources (see below) folder.
/themes/theme-name/layouts Layouts in Respond define the structure of pages for the application. They are written in HTML with special mustache-syntax based tags that allow the page to interact with the CMS. Learn more about Respond's Mustache Syntax.
/themes/theme-name/pages The pages define the default content for the about, blog, contact, error, home, and post pages that are created when you create a new site in Respond with the theme. You set which theme Respond uses by default in setup.php using the DEFAULT_THEME constant.
/themes/theme-name/styles The styles folder contains the LESS files that define the presentation for a site within Respond. The LESS files are compiled into CSS when the site is published or a change is made via the application.
/themes/theme-name/resources Version 2.9+: The resources folder holds any static files (e.g. images, fonts, JavaScript, or CSS libraries) that you don't want the user interacting with. This folder is copied to the /sites/[my-site]/themes/[theme-name]/resources folder and can be referenced with {{resources}} from the layout or ../themes/[theme-name]/resources/ from your LESS.

Layouts

The following layouts must be defined for each theme created in Respond. However, you can create as many layouts as you want for your theme.

Template File Description
layouts/blog.html Defines the default layout for the blog list page.
layouts/content.html Defines the default layout for all content pages. By default, these are any pages created in the pages/ directory.
layouts/home.html Defines the default layout for the home page.
layouts/post.html Defines the default layout for a blog post.

Styles

The following styles (LESS) files must be defined for each theme created in Respond. However, you can create as many LESS files as you want.

Respond uses LESS as a pre-processor for its CSS. LESS is great for template building and makes it easy for others to update the color scheme of your template. To learn what you can do with LESS, see the official documentation.

Styles File Description
styles/content.less Defines the presentation for content pages created in Respond.
styles/global.less Defines the global presentation for pages created in Respond. In the simple theme, all other LESS files import this stylesheet. However, you can decide how you want to structure these files.
styles/home.less Defines the presentation for the home page in Respond.

Pages

Respond allows you to define the content for the pages that are added by default when a site is created in Respond. The following pages are created by default in Respond 2.5+. You cannot define additional pages. You set which theme Respond uses by default in setup.php using the DEFAULT_THEME constant.

Page Description
pages/about.html This is just a simple about us page.
pages/blog.html A page that displays a list of recent blog posts ordered by the date created.
pages/contact.html A form to allow a visitor to the site to contact the site owner.
pages/error.html The page that is shown when a 404 (or other) error occurs on the site.
pages/home.html The home page of the site.
pages/post.html The sample blog post that is created with the site.

Managing Resources

Themes come with a lot of stuff (Images, CSS libraries, fonts, JavaScript libraries, CSS libraries, etc.) to make them work. While it makes sense to make some of this stuff accessible to the user via layouts, styles, and files. Some files are really just static to the theme and should not be accessible to the user. The resources folder provides a place to store these files and copy them to the site.

For layouts, resources are accessible via the new {{resources}} syntax. This provides you a quick way to reference your resources.

For your styles, there is no built-in way to reference your resources. Since styles, sit at the same depth throughout the app you can reference them via a relative url, e.g. ../themes/theme-name/resources/, or you can use a LESS variable, as shown below. 

@resources: '../themes/theme-name/resources/';
...
@import url("@{resources}sample.css");
...
background: transparent url("@{resources}sample.png") no-repeat 0 0;

Branding Colors

Respond allows you to expose colors through the Branding screen via LESS variables.

// Defining a color in LESS, will show up in colors as 'Background Color'
@background-color: #888888;

Setting a name and Description

The name and description for your theme is shown in the Theme menu item within Respond. To configure this name and description, simply update the theme.json file in your theme folder.

{
	"name": "Simple",
	"desc": "This is the default theme for Triangulate CMS and a great starting point for any responsive site"
}

Installing and Distributing your Theme

To install a theme, simply copy that theme to the /themes folder. Themes have no dependencies, so you can simply ZIP the theme folder to distribute it.
