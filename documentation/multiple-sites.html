Managing Multiple Sites

A core feature of Respond is the ability to create and manage multiple sites using the CMS.  This works out-of-the-box without any additional tools or plugins installed.   In this documentation, we will take a look at how this works and provide some guidance on how to manage multiple sites within the CMS

How it works

Sites in Respond are created "on an island".  What this means is that all users of the site can only access that site.  This also means that emails (or user logins) are not unique in the system.  They are unique per site.

Site Administrator

The one exception to this rule is the Site Administrator.  This property of a user allows him/her with the ability to view all sites created in the system and perform common functions.  See table below. 

Property Function
Domain Same as in the Settings.  The Domain is the URL where the site can be accessed publicly.  It also provides CORS access so that the site can access the REST API.
Amazon S3 Bucket The Amazon S3 bucket where files are stored (if applicable).
Status The status of the site:  Trial, Active, Expired.  Trial sites will see a message asking them to upgrade.
User Limit The number of users that can be created for the site.
File Limit The storage limit for the site (10MB, 100MB, 1GB, 10GB).

Any user in the system can become a Site Administrator.  This can be done only if you have access to the database.  You must navigate to that user in the database and set the SiteAdmin flag for the User to 1.
